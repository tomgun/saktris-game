# F-0019: Mobile View

## Acceptance Criteria

### AC1: Responsive Layout
- Game detects screen width < 800px as "mobile mode"
- Mobile layout: vertical stack (status → board → controls)
- Desktop layout: horizontal (board left, panel right)
- Margins reduce from 20px to 8px on mobile
- Layout switches dynamically on window resize

### AC2: Touch Input Support
- Tap square to select piece (same as click)
- Tap destination to move (same as click)
- Drag piece to move (same as mouse drag)
- Long-press (500ms) to start arrow drawing (replaces right-click)
- Mouse input still works on desktop

### AC3: Simplified Mobile UI
- Touch targets minimum 44px
- Compact status bar at top
- Piece queue horizontal below board (not side panel)
- Full-width buttons at bottom
- No horizontal scrolling required

### AC4: Works on Real Device
- Tested on mobile browser (iOS Safari / Android Chrome)
- No zoom issues (viewport meta tag set)
- Game playable end-to-end on phone

### AC5: Board Size Optimization (Portrait)
- Board should be as large as possible in portrait mode
- Other UI elements (status, controls) fit above/below board
- Maximize playable area on small screens

### AC6: Landscape Mode Support
- Pieces remain correctly positioned when rotating to landscape
- Game remains playable after orientation change
- Layout adapts appropriately to landscape dimensions
